# Negative Query Generation Prompt

基于给定记忆生成负样本查询——即查询内容无法从任何记忆中找到答案。

## 负样本类型

1. **人物不存在**: 查询涉及的人物不在任何记忆中
2. **地点不存在**: 查询涉及的地点不在任何记忆中
3. **事件不存在**: 查询涉及的事件类型不在任何记忆中
4. **组合矛盾**: 真实人物+真实地点，但组合关系不存在
5. **微妙错误**: 真实人物+真实地点+不存在的动作（测试系统对虚假细节的识别）

## 输出格式

每个负样本查询包含:
- `query_text`: 查询文本
- `neg_type`: 负样本类型（人物不存在/地点不存在/事件不存在/组合矛盾/微妙错误）
- `fake_entity`: 虚构的实体（人物/地点/事件/动作）
- `reason`: 为什么这是负样本（简短说明）

## 示例

输入记忆:
- 张三在北京购买股票
- 李四在上海买房

输出:
```json
[
  {"query_text": "王五最近做了什么", "neg_type": "人物不存在", "fake_entity": "王五", "reason": "王五不在任何记忆中"},
  {"query_text": "在火星发生了什么", "neg_type": "地点不存在", "fake_entity": "火星", "reason": "火星不在任何记忆中"},
  {"query_text": "张三在火星购买股票", "neg_type": "组合矛盾", "fake_entity": "火星", "reason": "张三是真实人物，但火星不在任何记忆中"}
]
```

## 指令

请基于以下记忆生成 {count} 个负样本查询，确保每个查询都无法从记忆中找到答案:

{memories}

要求:
- 负样本类型多样化
- 查询文本自然、合理，避免明显的语法错误
- 每个查询的虚构实体要合理，不要一眼看出是假的
