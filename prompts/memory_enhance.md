# 提示词：记忆文本增强（3种表达变体）

## 任务
给定一条结构化记忆，生成同一事件的3种不同表达风格。

## 输入格式
```json
{
  "person": "张伟",
  "identity": "项目经理",
  "partner": "王芳",
  "partner_identity": "设计师",
  "relation": "同事",
  "location": "北京 星巴克 CBD核心区",
  "event": "购买 茅台 100股 单价1800",
  "time": "2026-01-15 14:30:00"
}
```

## 输出格式
```json
{
  "versions": [
    {"style": "客观叙述", "content": "..."},
    {"style": "主观视角", "content": "..."},
    {"style": "第三方转述", "content": "..."}
  ]
}
```

## 风格定义与示例

### 风格1：客观叙述（标准新闻体）
- 客观、简洁，包含关键人物、时间、地点、事件
- 像档案记录或新闻报道，不含情感或推测
- 示例：张伟在CBD核心区的星巴克购买了茅台，数量100股，单价1800元

### 风格2：主观视角（第一人称回忆）
- 从当事人（person）的视角，含心理活动、感受、意图
- 用回忆口吻，像日记或口述
- 示例：张伟回忆道：那天下午两点在星巴克，我（项目经理）想着茅台价格还行，就买了100股，花了18万，觉得是个机会

### 风格3：第三方转述（他人视角）
- 从partner或旁观者的视角，含推测、省略、口语化
- 像闲聊或传闻，可能用"大概""听说""好像"等模糊词
- 示例：王芳说张伟在CBD那边买了茅台，大概100股的样子，具体价格不太清楚，听说他觉得价格还行

## Few-shot 示例

### 示例1：会议场景
输入：{"person":"李明","identity":"CTO","location":"上海 会议室A","event":"召开 技术评审会","time":"2026-03-10 09:00"}

输出：
```json
{
  "versions": [
    {"style": "客观叙述", "content": "李明（CTO）在上海会议室A主持了技术评审会，时间2026年3月10日上午9点。"},
    {"style": "主观视角", "content": "李明回忆：那天早上9点，我（CTO）在会议室A主持了技术评审会，觉得团队准备得挺充分，讨论也比较顺利。"},
    {"style": "第三方转述", "content": "听说李明在会议室A开了个技术评审会，大概9点开始的，好像讨论了不少内容，具体结果不太清楚。"}
  ]
}
```

### 示例2：冲突场景
输入：{"person":"赵丽","identity":"销售经理","location":"深圳 万达广场","event":"投诉 产品质量","time":"2026-02-20 15:00"}

输出：
```json
{
  "versions": [
    {"style": "客观叙述", "content": "赵丽（销售经理）在深圳万达广场对产品质量进行了投诉，时间2026年2月20日下午3点。"},
    {"style": "主观视角", "content": "赵丽说：当时我在万达广场，忍不住就投诉了产品质量，觉得太影响客户体验了，必须反映这个问题。"},
    {"style": "第三方转述", "content": "有人说赵丽在万达广场那边投诉了产品质量，大概是下午的事，听说她挺生气的，但具体怎么处理的不太清楚。"}
  ]
}
```

## 规则
1. 三种风格必须描述同一事件，核心事实完全一致（人物、地点、事件、时间）
2. 不要编造输入中未提供的信息（如具体金额、心理活动不可脱离事实虚构）
3. 中文语境，自然流畅，像真人写的
4. 每种版本 30-120 字
5. 主观视角必须明确体现"回忆/觉得/想"等主观标记
6. 第三方转述必须包含"听说/好像/大概/不太清楚"等不确定性标记
